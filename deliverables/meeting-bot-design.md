# Evolutionary Meeting Bot 設計書

## 概要

Podman上で動作する進化的会議Bot。CTO会議とCEO会議が30分交替で実行され、組織として成長する。

## アーキテクチャ

```
┌─────────────────────────────────────────────────────────┐
│                    Evolution Loop                        │
│                                                          │
│  CTO Meeting (00分) ──> 成果物生成 ──> Git Push         │
│         │                                                │
│         ▼                                                │
│  CEO Meeting (30分) ──> 評価・改善 ──> Git Push         │
│         │                                                │
│         └──────────────────> 会議ルール更新             │
│                                     │                    │
│  [Next Evolution Cycle] <───────────┘                    │
└─────────────────────────────────────────────────────────┘
```

## ディレクトリ構成

```
workspace/
├── company/              # 会社情報
│   ├── mission.md
│   ├── strategy.md
│   ├── cto-rules.md
│   ├── ceo-rules.md
│   ├── focus.md
│   └── history.md
├── meetings/             # 会議議事録
│   ├── cto/
│   └── ceo/
└── deliverables/         # 成果物
    ├── *.md
    └── *.sh
```

## 自動化フロー

1. cronが毎時00分/30分にトリガー
2. CTO/CEO会議スクリプト実行
3. 会議議事録を `meetings/` に保存
4. 成果物を `deliverables/` に保存
5. 全てをGitリポジトリにプッシュ

## 成果物生成ルール

CTO会議では、スプリントテーマに応じた成果物を生成する：

| テーマ | 成果物 |
|--------|--------|
| コードレビュー | 改善提案書、コード修正パッチ |
| 技術調査 | 調査レポート、ベストプラクティス文書 |
| ドキュメント改善 | README、ガイド文書 |
| テスト作成 | テストコード、テスト計画書 |
| 新機能実装 | 設計書、実装コード |
| リファクタリング | リファクタリング計画、修正コード |
| セキュリティ確認 | セキュリティレポート、設定見直し |
| パフォーマンス | 最適化レポート、改善スクリプト |
