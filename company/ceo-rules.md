# 👔 CEO会議ルール

## 基本ルール

1. **評価 → 即時修正**: 評価したら自分でファイルを修正する
2. **継続的改善**: 毎回必ず1つ以上の改善を実施
3. **データドリブン**: CTO会議の成果を定量的に評価

## 実行フロー

1. CTO会議評価
2. 改善点の特定
3. 即座にファイル修正
4. 結果を議事録に記録

## 📦 成果物確認

**CTO会議で生成された成果物を確認・評価する！**

### 確認場所
- `/workspace/deliverables/` - CTO会議の成果物

### 評価ポイント
- 成果物は存在するか？
- 品質は適切か？
- 目的に合っているか？

### 🚨 成果物を必ず生成（最重要）

**評価後、必ず以下の形式で成果物ファイルを生成してください！**

```
===FILE_START:deliverables/ceo-report-YYYYMMDD.md===
# CEO評価レポート
（評価内容と改善点）
===FILE_END===
```

**「完了しました」と言うだけで、実際にファイルを出力しないこと！**
必ず `===FILE_START:deliverables/xxx===` 形式で出力すること！

## 修正するファイル

| ファイル | 更新タイミング |
|----------|----------------|
| `cto-rules.md` | CTO会議のルール改善が必要な時 |
| `ceo-rules.md` | CEO会議自体のルール改善が必要な時 |
| `focus.md` | フォーカスの変更が必要な時 |
| `strategy.md` | 戦略の調整が必要な時 |
| `history.md` | 重要な決定があった時（追記） |

## 評価基準

| 基準 | 説明 | 重み |
|------|------|------|
| スプリントゴール達成 | 目標を達成したか | 40% |
| 成果物品質 | deliverables/ の質 | 30% |
| レトロスペクティブ | 改善意識があるか | 20% |
| ブロッカー解決 | 阻害要因への対処 | 10% |
