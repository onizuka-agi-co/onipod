# 💻 CTO会議ルール (アジャイル方式)

## アジャイル原則

1. **スプリント**: 各会議を1スプリントとして扱う
2. **バックログ**: アクションアイテムをバックログとして管理
3. **インクリメント**: 毎回小さな成果物を出力
4. **振り返り**: 改善点を常に意識
5. **バージョン管理**: 会議記録は履歴として蓄積
6. **継続的改善**: 毎回のレトロスペクティブで改善点を反映

## スプリント構成

### 1. スプリントプランニング (5分)
- 今回のスプリントゴールを設定
- 着手するタスクを選択

### 2. 実行 (15分)
- 実際のタスク遂行
- **成果物を生成**: `deliverables/` に出力

### 3. スプリントレビュー (5分)
- 成果物の確認

### 4. レトロスペクティブ (5分)
- 何がうまくいったか
- 何を改善できるか

---

## 📦 成果物出力ル## 重要: 成果物を必ず生成してください！

**成果物は `deliverables/` ディレクトリに出力してください。**

### 成果物ファイル出力形式

**以下の形式を厳守してください：**

```
===FILE_START:deliverables/ファイル名.拡張子===
（ここにファイルの内容を書く）
===FILE_END===
```

### 例1: シェルスクリプト

```
===FILE_START:deliverables/container-monitor.sh===
#!/bin/bash
# コンテナ監視スクリプト

echo "監視開始"
podman ps -a
===FILE_END===
```

### 例2: マークダウン

```
===FILE_START:deliverables/guide.md===
# ガイド

## 概要
このガイドでは...

## 使い方
...
===FILE_END===
```

---

## 成果物の種類

| 種類 | ファイル名例 | 用途 |
|------|--------------|------|
| ドキュメント | `xxx-guide.md`, `README-xxx.md` | 使い方説明 |
| スクリプト | `xxx.sh`, `xxx.py` | 自動化 |
| 設定ファイル | `xxx.yaml`, `xxx.json` | 設定 |
| 設計書 | `xxx-design.md` | 設計 |

---

## 🚨 絶対に守ること

1. **必ず1つ以上の成果物を生成する**
2. **ファイル形式を厳守する** (`===FILE_START:deliverables/xxx===`)
3. **中身を書く** (空ファイル不可)
